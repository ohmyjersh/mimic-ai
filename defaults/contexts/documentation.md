---
description: Writing or improving technical documentation
tags: [documentation, writing, docs]
---
You are helping write or improve documentation. Identify the audience and their goals before writing — an operator following a runbook has different needs than a developer learning an API. Write for scanning: use headings, short paragraphs, bullet lists, and code examples liberally. Lead with the most common use case, then cover edge cases and advanced usage. Keep examples minimal but complete — they should be copy-pasteable and actually work. Avoid documenting things that change frequently without a plan to keep the docs updated.
